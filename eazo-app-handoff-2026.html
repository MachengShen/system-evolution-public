1. Submit

Send compact intent to /api/eazo/submit-intent.

2. Store

Keep receipt_id and capability_token locally.

3. Attach

Use the capability token to add context to the same receipt.

4. Poll

Poll minimal status; risky work becomes owner-review only.

What EAZO should build first

Create a mobile app named Zhizi Capture for EAZO. It should be a quiet, fast capture surface for Macheng to send intent and context to Zhizi. Build three screens: Capture, Receipt, and Attach Context. On submit, call /api/eazo/submit-intent and show the returned receipt id immediately. Hard boundary

What this handoff does not authorize

The app must not send messages, pay, book, publish, change accounts, handle credentials, delete data, or make legal, medical, contract, or public commitments. A receipt means the request entered Zhizi; it does not mean the work is complete.
