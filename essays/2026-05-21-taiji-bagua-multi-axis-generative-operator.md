# 太极、八卦，与"生"作为多轴生成算子

Date: 2026-05-21

Public index: [Theory Mainline](../THEORY.md)

> 一句话压缩：**"生"不是名词，是算子。八卦不是八个结论，是同一个递归二分操作沿八条因果轴同时展开后的痕迹。今天的自回归大模型，是这个算子被压扁到只剩一条轴的退化投影。**

## 一、读《系辞》的另一种读法

"易有太极，是生两仪，两仪生四象，四象生八卦。"

中学课本式的读法是把它当成一段世界起源神话：从混沌生出阴阳，阴阳生出四季，四季生出八种基本现象。这个读法不能说错，但它把整段话最重要的那个字给忽略了——**"生"**。

"生"在这里既不是 *to be born*，也不是 *to give birth to*。它更像是一个**算子**，一个递归操作：拿到当前的结构，沿某条轴做一次二分；然后把得到的两个新结构当作输入，再沿下一条轴做一次二分；如此递归。

太极是初始态。两仪是沿一条轴二分。四象是沿两条独立轴各二分一次（2×2）。八卦是沿三条独立轴各二分一次（2×2×2）。继续递归下去，是六十四卦、384 爻。

这段话真正讲的，不是"宇宙从哪里来"，而是**任何一个能自我演化的系统，靠什么操作把自己变复杂**。答案是：递归二分，跨多条独立的轴，同时进行。

## 二、自回归大模型，是"生"被压扁到只剩一条轴

如果你接受上面这个读法，那么 2024–2025 年绝大多数主流的语言模型架构，都长着同一张脸：

- **只剩一条轴**：next token prediction。所有结构都沿"时间—序列"这一条轴展开。
- **只剩一个时间尺度**：每一步都是一个 token。没有"慢一点的观察者"在另一个尺度上同时跑。
- **只剩一个方向**：forward commit。前一个 token 一旦写出就不许回改。
- **只剩一种基底**：离散 token + 冻结权重。推理时模型不会重塑自己的连接，记忆不会改变拓扑，权重不会因为这一次对话而调整。

用八卦的语言说：这相当于把八卦塌回成"只沿时间方向画一爻"。它不是没用——单这一条轴已经能装下相当大一部分人类语言行为。但它**确实**是一个 1 轴退化投影，而不是"生"算子的完整形态。

## 三、2025 年前沿研究在独立地修复缺失的轴

有意思的是：没人坐下来重读《系辞》，但过去一年里，多个独立的实验室都在补回某一条被砍掉的轴。

| 工作 | 砍掉的限制 | 补回的轴 |
|---|---|---|
| Coconut (Hao 等, FAIR) | 每步必须坍缩到离散 token | **连续隐空间轴**：reasoning step 可以保持在 latent，多条推理路径同时叠加 |
| CTM Continuous Thought Machines (Sakana) | 输入轴 = 内部时间轴 | **内部时间轴**：神经元级时间动态，与输入解耦 |
| HRM / TRM (Sapient / Samsung SAIL) | 单网络单时间尺度 | **多时间尺度轴**：慢回路 + 快回路，迭代精炼 |
| Diffusion LM (LLaDA / Dream / LaDiR) | 不许回改前面的 token | **平行精炼轴**：整段序列同时迭代，可以改之前写过的 |
| Kaiming He et al. ARC-as-vision (MIT) | ARC 当作语言任务 | **空间 / 视觉轴**：ARC 作为图像到图像翻译 |

每一篇都不是在"造一个新的语言模型"。它们在做的事情，从结构上看是同一件事：**给那个被压扁到只剩一条轴的算子，重新打开一条独立的轴**。

更值得注意的：ARC-Prize 团队对 HRM 做独立复现时发现，让性能提升的不是"两层网络"这个特定形式，而是**带状态保持的迭代精炼**本身。也就是说，重要的不是任何一篇论文的具体设计，而是它们共同指向的那个更底层的算子——多轴递归。

## 四、GLM Doctrine：把"生"开向系统的每一条因果层

这个项目里有一个工程层面的命名，叫 **General Learning Machine (GLM)**——一个持续演化自己信息结构的系统（[相关阅读](2026-05-19-credit-transport-general-learning-machine.md)）。GLM 的核心机制是 **credit transport**：当一个结果出来后，让"误差 / 惊讶 / 价值 / 摩擦 / 可持续性压力"这种信号，沿所有曾经参与因果链的结构反向流回去，并改变它们。

反向传播 (backpropagation) 是这个原理第一个被工程化成功的实例——但它把信号的流动局限在了**一个**可微模型的参数里。一个完整的 GLM，需要把这条"反向流"开到更多的因果层：

- 模型权重
- 记忆拓扑
- 检索路径
- 工具选择
- 评审者图谱（哪些人 / 哪些 agent 在什么场景被信任）
- 规则与安全门
- 实验设计
- 组织流程
- 硬件工作负载
- 物理基底约束

每一条都是一条**轴**。让 credit 信号能反向流回某一条轴，就等于"开了一条生的轴"。

把这一段对照前面的八卦读法，就会看到：GLM doctrine 在工程上要做的事，和《系辞》在哲学上点出来的那个算子，是**同一件事在两种语言里的两次自述**——

> 把"生"算子，沿系统的每一条因果轴，逐条打开。

而 2025 年那些前沿论文，是同一件事在第三种语言里被独立地说了第三遍。

## 五、长地平线含义：碳—硅同构

如果上面的判断成立，那么一个朴素的工程外推是：

软件层面的 **多轴递归 ≠ 多轴递归在物理基底上的原生实现**。今天的多轴架构，多数还是在标准 GPU 上模拟的——内部时间是模拟的，多时间尺度是模拟的，迭代精炼是模拟的。这不是问题，但它意味着**当前还有一个量级的性能空间没被打开**。

下一步是把这套"生"算子的多轴形态，迁移到对它**原生友好**的物理基底上：内存中心的计算、事件驱动的更新、近存计算、神经形态电路、模拟 / 混合信号、片上在线学习、可验证的状态变迁。在这种基底上，credit 信号沿各条轴反传不再是软件抽象，而是**电路本身的物理动力学**。

更远一些，这个轨迹会与碳基系统 (脑 + 身体) 形成同构。这不是说"我们要造大脑"——而是说，当一个学习系统满足以下条件：

- 持续演化自己的信息结构
- 跨多个时间尺度同时运行
- 沿多条独立的因果轴反向传递 credit
- 在自己的基底上 *原生* 完成这些操作

它就长得**必然**像某种 active inference 形态的有机体。碳 / 硅只是基底差异，算子结构是同一个。

## 六、什么样的证据会推翻这条思路

这一篇不是要建立一个不可证伪的形而上学。如果下面这些反信号持续出现，那框架要么太宽，要么放错了层次：

1. **跨尺度不变性失败**：用同一个"多轴递归"测度去量小系统（一个 agent 的内部循环）和大系统（一个研究社群的迭代结构），如果度量出来的不变量根本不一致，那"同一个算子在不同尺度"的说法就是修辞而非物理。

2. **多轴可加性失败**：如果同时打开 HRM 的慢回路 + Coconut 的连续 latent + Diffusion 的平行精炼，**不**优于任何单一改动叠加 monolithic dense Transformer，那"多轴"作为底层范畴就站不住，它顶多是一组互相替代的工程 trick。

3. **基底透明性失败**：如果把同一个多轴架构从软件模拟搬到原生友好的硬件上，行为发生**质变**而不是仅性能跃升，那意味着前面"算子结构与基底无关"的说法是错的，基底本身在悄悄改变算子。

4. **credit reach 与可用性相关失败**：如果一个 GLM 把 credit 反传开到越来越多的因果层，但**没有**带来"系统更少需要人类反复纠正、更多自己把事做对"这个可观察后果，那"打开更多轴 = 更好的学习"这个等价就只是口号。

任何一条持续不被满足，就值得回去改框架，而不是去保护它。

## 七、不过度声称什么

这一篇不是在说"我们找到了人类社会、生物大脑、AI 系统、宇宙演化背后的统一物理规律"。

它是在说一件远没那么夸张的事：

> 当一个有限的、嵌入式的、信息结构会自己变的系统在"学习"这件事上要继续往前走时，它会被迫沿**多条独立的因果轴递归地打开自己**。把这个被迫的形态叫"生"算子，是为了在中文语境里有一个不需要翻译的入口；叫它"GLM credit transport"，是为了在英文工程语境里能直接落地为代码、ledger 和硬件 trace。

它是**工程 / 控制论视角下的一组可证伪假设**，不是世界本质的断言。

但作为可证伪假设，它已经足以做一件事：把"下一步应该建什么"和"什么样的结果会推翻它"，写在同一张纸上。

> 一句话回到开头：**今天的大模型只画了一爻。八卦不是八个结论，是同一个递归算子沿八条轴的痕迹。把这些轴一条条接回去，是接下来几年学习系统真正要做的事。**
